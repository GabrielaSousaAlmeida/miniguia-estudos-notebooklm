# README – Projeto de Estudos QA com IA (NotebookLM)

##  Contexto e Objetivos

Este repositório foi criado como parte do meu processo de aprendizado na área de **Quality Assurance (QA)**, com foco em acelerar minha entrada no mercado de trabalho.

O estudo foi conduzido com apoio do **NotebookLM**, utilizando inteligência artificial para organizar conteúdos, gerar insights e estruturar conhecimento.

###  Objetivos principais:
- Consolidar conhecimentos em **testes manuais**
- Aprender e praticar **testes de API com Postman**
- Desenvolver habilidades iniciais em **automação com Cypress e Robot Framework**
- Compreender na prática conceitos de **metodologias ágeis**
- Criar uma metodologia de estudos eficiente e reaplicável

### Aprendizados (Cicatrizes)

Durante o processo, identifiquei alguns desafios importantes:

- Prompts muito genéricos geravam respostas superficiais  
- Falta de contexto diminuía a qualidade das respostas  
- Nem sempre a IA traz exemplos práticos

---

###  Melhorias aplicadas

- Comecei a usar prompts mais específicos  
- Passei a pedir exemplos práticos sempre  
- Dividi perguntas grandes em partes menores

##  Miniguia de Estudo (Entrega Final)

### 1. Resumos Estruturados

#### Testes Manuais
- Validação do sistema sem uso de automação  
- Foco em encontrar bugs e validar regras de negócio  

**Tipos:**
- Teste funcional  
- Teste exploratório  
- Teste de regressão  

---

#### Testes de API (Postman)
- Permite testar requisições HTTP (**GET, POST, PUT, DELETE**)  

**Validação de:**
- Status code (200, 404, 500)  
- Tempo de resposta  
- Corpo da resposta (JSON)  

---

#### Cypress (Automação)
- Ferramenta de testes automatizados para front-end  
- Usa **JavaScript**  
- Executa testes no navegador  
- Ideal para testes **E2E (end-to-end)**  

---

#### Robot Framework
- Framework de automação baseado em palavras-chave  
- Mais simples para iniciantes  
- Pode ser usado com **Python**  

---

#### Metodologias Ágeis (Scrum)
- Trabalho em ciclos (**Sprints**)  

**Reuniões:**
- Daily  
- Planning  
- Review  

- QA atua validando entregas continuamente  

---

## 2. Glossário

- **Bug** → erro no sistema  
- **Teste de regressão** → verifica se algo antigo quebrou  
- **API** → comunicação entre sistemas  
- **Automação** → testes feitos por código  
- **Sprint** → ciclo de trabalho no Scrum
